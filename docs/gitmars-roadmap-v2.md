# Gitmars 升级路线 2.0

> 重新审视设计初衷，打造真正有价值的 Git 工作流工具

---

## 一、设计初衷回顾

### 1.1 核心使命

```
┌─────────────────────────────────────────────────────────────┐
│                     Gitmars 的使命                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   让每个开发者都能轻松驾驭 Git                               │
│                                                             │
│   • 降低学习成本 —— 不懂 Git 原理也能高效协作                │
│   • 减少操作步骤 —— 复杂流程一键完成                         │
│   • 避免人为错误 —— 最佳实践内置，想错都难                   │
│   • 提升团队效率 —— 统一工作流，减少沟通成本                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 当前问题反思

| 问题 | 现状 | 影响 |
|------|------|------|
| **定位模糊** | 像 Git 命令的"包装器" | 用户觉得"我直接用 git 也行" |
| **价值不够独特** | 主要是简化命令 | 缺乏"离不开"的理由 |
| **学习成本仍在** | 仍需理解分支模型 | 新人上手有门槛 |
| **智能化不足** | 依赖用户决策 | 没有真正"降低"难度 |
| **生态缺失** | 独立工具 | 无法融入开发流程 |

### 1.3 重新定义 Gitmars

```
从: Git 命令封装工具
到: Git 工作流智能助手

从: 简化命令输入
到: 让用户不需要思考 Git

从: 工具属性
到: 平台 + 生态
```

## 三、核心能力升级

### 3.1 智能化 —— 让工具主动帮用户

**现状**：用户需要知道用什么命令
**目标**：工具告诉用户应该做什么

```shell
# 现在：用户主动
gitm start feature login
gitm update
gitm combine -d

# 未来：工具主动
gitm
> 检测到您在 feature/login 分支有 3 个未推送的提交
> 检测到 develop 有 5 个新提交需要同步
> 建议执行: gitm update && gitm combine -d
> 是否执行？[Y/n]
```

**智能场景**：

| 场景 | 智能建议 |
|------|----------|
| 分支落后上游 | "您的分支落后 develop 5 个提交，建议执行 gitm update" |
| 有未推送提交 | "您有 3 个提交未推送，是否推送？" |
| 长时间未同步 | "您已 7 天未同步上游代码，建议更新" |
| 冲突风险 | "检测到 3 个文件可能冲突，建议先沟通" |
| 分支可清理 | "发现 5 个已合并分支可清理，释放空间" |

### 3.2 引导式 —— 让新手无感上手

**现状**：需要记住命令和参数
**目标**：交互式引导，像对话一样自然

```shell
# 现在：需要知道命令
gitm start feature login

# 未来：交互式引导
gitm
> 您想做什么？
  ❯ 开始新功能开发
    修复 Bug
    发布代码
    查看状态
    其他操作...

> 您想开发什么功能？
  登录功能
  > 请输入功能名称: login

> 正在创建分支 feature/login...
> ✓ 分支创建成功！
>
> 接下来您可以：
  ❯ 开始编码（我会提醒您后续步骤）
    查看开发指南
    返回主菜单
```

### 3.3 自动化 —— 让流程自动流转

**现状**：每步都需要手动执行
**目标**：设定目标，自动完成

```shell
# 现在：手动执行每一步
gitm start feature login
# ... 编码 ...
gitm update
gitm combine -d
gitm combine -p
gitm end

# 未来：目标驱动
gitm flow "完成登录功能开发"
> 目标分析：创建分支 → 开发 → 测试 → 发布 → 清理
>
> [1/5] 创建分支 feature/login ✓
> [2/5] 等待开发完成...
>       检测到代码变更，是否提交？[Y/n]
>       检测到测试通过，是否合并到测试环境？[Y/n]
> [3/5] 合并到 develop ✓
>       测试环境部署中... ✓
>       测试通过，是否发布？[Y/n]
> [4/5] 合并到 bugfix ✓
> [5/5] 清理分支 ✓
>
> 🎉 登录功能开发完成！
```

## 六、v8.5.0 - 智能化版本

**发布目标**：让工具主动帮助用户，真正降低学习成本

### 6.1 智能建议系统（核心）

**设计理念**：用户不需要知道用什么命令，工具告诉用户

```typescript
// 智能建议引擎
interface Suggestion {
    id: string
    priority: 'high' | 'medium' | 'low'
    message: string
    action: string
    autoFix?: boolean
}

class SuggestionEngine {
    // 分析当前状态
    analyze(context: GitContext): Suggestion[]

    // 持续监控
    watch(callback: (suggestions: Suggestion[]) => void): void

    // 自动执行
    autoExecute(suggestion: Suggestion): Promise<void>
}
```

**建议规则示例**：

```typescript
const rules: SuggestionRule[] = [
    {
        id: 'branch-outdated',
        condition: ctx => ctx.behind > 0,
        message: '您的分支落后上游 {n} 个提交',
        action: 'gitm update',
        priority: 'high'
    },
    {
        id: 'unpushed-commits',
        condition: ctx => ctx.ahead > 3,
        message: '您有 {n} 个提交未推送，建议及时推送',
        action: 'git push',
        priority: 'medium'
    },
    {
        id: 'ready-to-combine',
        condition: ctx => ctx.hasUnpushed && !ctx.hasConflict,
        message: '代码已提交，可以合并到测试环境',
        action: 'gitm combine -d',
        priority: 'medium'
    },
    {
        id: 'branch-cleanup',
        condition: ctx => ctx.mergedBranches.length > 5,
        message: '发现 {n} 个已合并分支可清理',
        action: 'gitm cleanbranch',
        priority: 'low'
    }
]
```
